# 古琴教学点管理系统

这是一个面向古琴教学点的管理系统，覆盖售前线索、体验课、正式排课、课后复盘、AI 学员评估、购课续课、购琴进度、物料领取、考核荣誉和活动参与等完整学员生命周期。

## 项目目标

系统要解决三个核心问题：

1. 教学管理：老师排课、上课记录、课后复盘、老师课量统计。
2. 学员管理：售前跟进、体验课、正式学员、学习进度、考核荣誉、活动参与。
3. 经营管理：购课、续课、购琴、教务跟进任务、AI 辅助判断成交机会和流失风险。

## 用户角色

- 管理者：拥有全部权限。
- 教务老师：管理全部学生、售前、购琴、物料、活动、考核、排课和跟进任务。
- 普通老师：只能查看自己的课程、自己的学生、自己的上课记录，并填写课后复盘。

## 第一阶段 MVP

第一阶段优先实现：

- 用户登录与角色权限
- 学生档案
- 售前跟进
- 体验课排课
- 正式课排课
- 教室管理
- 购课记录与课时余额
- 老师上课记录
- 课后复盘
- AI 学员评估
- 教务跟进任务
- 购琴进度
- 老师课量统计
- 入学礼包领取
- 初中高级考核
- 胸章发放记录

活动管理可以作为第二阶段增强。

## 关键业务规则

- 老师不填写课后复盘，课程不能完成。
- 课程未完成，不扣课。
- 体验课也占用老师和教室时间。
- 同一时间，一个教室只能安排一批学生。
- 同一时间，一个老师只能上一节课。
- AI 只给建议，不自动修改学生状态。
- 教务确认后，再执行续课、购琴、流失挽回等跟进动作。
